Data Analysis Application for Agriculture

Posted on: July 09, 2022

The application is being developed incrementally, and new features are continuously added. I will briefly explain what it entails.

The aim of the application is to provide a set of statistics for agricultural producers. The company has equipment that gathers position and performance information from harvesters and planters.

The language used is Python. Libraries like Pandas, GeoPandas, and Shapely are used for data analysis, and Django for the API.

Main Algorithms

During the development of the application, I programmed a series of algorithms which I wrote about in this post, which may be interesting.

Main Features I Developed

Progress Reports Updated Every 8 Hours During the Current Harvest Season

These reports detail:

  • Area worked by the machines
  • Overlapped area
  • Percentage harvested in each farm
  • Yield map of each farm
  • Average yield of each farm
  • Average humidity of each farm
  • Detection of low yield areas in the farms
  • Error detection in the equipment
  • Machines used in each farm
  • Progress polygon with harvested area of each farm
  • Start and end date of harvest or planting for each farm

The calculation results can be viewed from a web platform or exported in various formats, such as: PDF, Excel, CSV, PNG, GeoJSON, etc.

API for Frontend Use

  • Information grouping by company
  • Detection of events requiring field team action
  • Management of machines and farms associated with each harvest season
  • Detection of farms from planting progress polygons
  • Uploading farm polygons from GeoJSON
  • Export in multiple formats
  • Availability of satellite images with various vegetation indices for each farm (NDVI, ReCL)

Work Status Detection Using Artificial Intelligence

A Machine Learning model was trained using the FastAI library to detect when a machine is working or not. Thanks to this, operational costs and work are saved as it only requires planters to have a GPS installed instead of one or more sensors.

This functionality also allows us to automatically detect problems in machines, such as disconnected, faulty, or misconfigured sensors, and send alerts to solve them.

Spatial Indices Using Earth Engine and Other Data Sources

I developed an integration with Google Earth Engine, enabling the retrieval of various vegetation indices such as NDVI for each field. This allows for a comparison of these data over time, helping to explain subsequent crop behavior and increasing the information available during analysis.

Complex Sensor Calibration System for Yield/Humidity

Develop a complex server-side system to calibrate the yield and humidity data from harvesters over time. Real data measured in the field using scales and humidity sensors is used as the baseline, and the data from this measurement is loaded into the system, allowing for retroactive calibration of the data, and generating a calibration curve that is used to correct the data from the harvesters. This allows the yield and humidity data to be more accurate, being able to adjust each individual harvester.

Final Comments

The solution provides the producer with a series of statistics that help optimize field usage, improving productivity, allowing them to prioritize those fields with better performance and identify problematic areas in the fields.
